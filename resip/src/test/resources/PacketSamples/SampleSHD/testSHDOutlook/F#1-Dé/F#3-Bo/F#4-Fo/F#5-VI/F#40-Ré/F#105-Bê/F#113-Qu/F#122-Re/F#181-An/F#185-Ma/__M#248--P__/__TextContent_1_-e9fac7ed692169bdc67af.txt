Résumé:                     [API-Ingests] : « ArchiveTransferReply » avec 
ReplyCode=KO, via SOAP UI, alors qu'il est OK via IHM !
Description: 
Via l’API la réponse est :
==> soit ATR , avec code KO 
==> soit http 4xx

En pièces joints les réponses avec dans leurs noms le paramétrage SOAP UI
utilisé.
Exemple content_type_application_zip indique que l'on a fait le test via SOAP UI
avec la balise content-type=application/zip

La documentation RAML mentionne pour un body les différent Media type
accéptés (notamment application/zip), or cela ne fonctionne pas !
Le paramétrage se fait normalement dans le Header
Ci-joint un copie écran du 1° paramétrage qui nous a été conseillé via
POSTMAN transmis copie_ecran_postman_conseille.png, or il renvoie un ATR avec
RC=KO,
