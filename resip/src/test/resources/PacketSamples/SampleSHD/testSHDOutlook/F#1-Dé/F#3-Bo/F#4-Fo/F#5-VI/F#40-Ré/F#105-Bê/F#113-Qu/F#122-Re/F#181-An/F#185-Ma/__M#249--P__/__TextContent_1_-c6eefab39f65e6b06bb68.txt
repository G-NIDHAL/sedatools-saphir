Résumé:                     Impossibilité d'obtenir une liste d'entrée dans
le journal des opérations via balise evDateTime
Description: 
Nous cherchons a obtenir une liste d'entrée dans le journal des opérations en
utilisant la balise evDateTime comme filtre.
Nous obtenons une erreur HTTP 500, erreur non documentée

Étapes pour le reproduire: 
Voila la requête utilisée : 
{
  "$roots": [],
  "$query": [
    {
      "$and": [
        {
          "$eq": {
            "evTypeProc": "INGEST"
          }
        },
        {
          "$and": [
            {
              "$gte": {
                "evDateTime": "2016-12-12T09:00:00.000"
              }
            },
            {
              "$lte": {
                "evDateTime": "2016-12-12T19:00:00.000"
              }
            }
          ]
        }
      ]
    }
  ],
  "$filter": {},
  "$projection": {}
}


Les headers sont configurés comme ceci : 
X-Http-Method-Override : Get
Content-Type : application/json
Accept : application/json

L'url est en POST
