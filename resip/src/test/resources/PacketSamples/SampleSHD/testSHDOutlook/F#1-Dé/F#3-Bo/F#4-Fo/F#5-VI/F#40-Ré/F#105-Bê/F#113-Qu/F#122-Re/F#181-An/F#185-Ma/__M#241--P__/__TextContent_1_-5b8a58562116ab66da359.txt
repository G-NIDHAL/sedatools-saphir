Résumé:                     Impossibilité d'utiliser l'opérateur $and dans
la collection Rules
Description: 
Nous cherchons a obtenir la liste des règles de type AccessRule et qui ont une
valeur 75 dans le champs RuleDuration via cette requête  : 
{
  "$query":
    {
      "$and": [
        {
          "$eq": {
            "RuleType": "AccessRule"
          }
        },
        {
          "$eq": {
            "RuleDuration": "75"
          }
        }
      ]
    }
,
  "$filter": {},
  "$projection": {}
}

Nous obtenons a chaque tentative une erreur 500
Nous avons essayé la même requête mais avec un "$or" à la place du "$and" et
nous obtenons bien comme résultat toutes les AccessRule et toutes les règles
avec une Duration de 75

Dans d'autre collection nous avons tester la balise "$and" avec succès :
{
  "$roots": [],
  "$query": [
    {
      "$and": [
        {
          "$match": {
            "Title": "api"
          }
        },
        {
          "$match": {
            "Description": "Vitam"
          }
        }
      ],
      "$depth": 20
    }
  ],
  "$filter": {
    "$orderby": {
      "TransactedDate": 1
    }
  },
  "$projection": {}
}
